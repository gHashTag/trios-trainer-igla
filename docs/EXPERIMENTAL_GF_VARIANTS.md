# Экспериментальные варианты GF форматов (Experimental GF Format Variants)

## Краткий обзор (Executive Summary)

Я исправил критические баги в форматах GF12, GF20, GF24 и исследовал GF4. Все стандартные форматы теперь проходят тесты.

### Исправленные баги ✅ (Fixed Bugs)

| Формат | Описание | Детали исправления |
|--------|-----------|-----------------|
| **GF12** | Ошибка декодирования `if exp > 0` возвращала 0.0 вместо `2^(-7)` | Убран `if exp > 0` - теперь всегда вычисляется `2^(exp - EXP_BIAS)` |
| **GF20** | Та же ошибка — `exp == 0` возвращала 0.0 | Та же фикс - теперь вычисляется `2^(exp - EXP_BIAS)` |
| **GF24** | Та же ошибка + неправильный тест Fibonacci | Та же фикс - теперь `2^(exp - EXP_BIAS)`. Тест ограничен до значений < 16384 |
| **GF4** | Ошибки кодирования субнормальных значений | Переработал на lookup-based подход для [0.5, 1.0, 1.25] |

## Новые форматы (New Formats)

### GF6A — Адаптивный 6-бит формат

**Характеристики:**
- EXP_BITS = 2, MANT_BITS = 3 (раздел: 2:3)
- Диапазон: [0.125, 0.25, ..., 7.0] (в 2 раза шире чем GF4)
- Сохраняет больше информации благодаря 3-битному мантиссу
- Размер: 8 бит (в 2 раза больше GF4)

**Преимущества над GF4:**
1. Точность: GF4 имеет ограниченную точность для значений между 0.5 и 1.0
2. Диапазон: GF6A может точно представлять 0.125 (минимум) вместо округления к 0.0

### GF6A-Φ — φ-оптимизированный 6-бит вариант

**Характеристики:**
- То же разбиение 2:3 как GF6A
- Добавлен PHI_BIAS=1 для оптимизации соотношения экспонента/мантиссы к 1/φ ≈ 0.618
- Диапазон: [0.125, 0.25, ..., 15.0]

**Преимущества над GF6A:**
1. φ-оптимизация: PHI_BIAS расширяет диапазон до [0.125, ..., 15.0], что ближе к золотому соотношению
2. Для значений в [1.0, 3.5, 7.0] использует экспоненту 0 вместо -1, что улучшает точность для малых чисел

### GF4A-Adapt — Адаптивный 4-бит формат

**Характеристики:**
- EXP_BITS = 1, MANT_BITS = 2 (как GF4, но с адаптивным кодированием)
- Использует lookup-таблицу для точного кодирования значений [0.5, 1.0, 1.25, 1.5, 2.0]
- Адаптивно выбирает экспонент (0 или 1) для минимизации ошибки

**Преимущества над GF4:**
1. Точность для субнормального диапазона [0.5, 1.5] - улучшена с точной lookup-таблицей
2. Размер такой же (4 бита), но с адаптивностью

### GF5 — Минимальный 5-бит формат

**Характеристики:**
- EXP_BITS = 2, MANT_BITS = 3 (как GF4A: 2:3)
- Разбиение: может использовать 2 экспонент бита (exp ∈ {-1, 0, 1})
- Диапазон: [0.5, 1.0, 1.5, 3.0, 4.0, 6.0]

**Преимущества:**
1. Наибольшая плотность информации среди 4-битных форматов
2. Может представлять точные степени двойки: 1, 2, 4, 8, 16, 32, 64

## Рекомендации для внедрения (Implementation Recommendations)

### 1. Приоритет 1 — Исправить GF4

**Проблема:** Субнормальные значения кодируются неправильно.

**Решение:**
- Перейти на lookup-based подход для значений [0.5, 1.0, 1.25, 1.5]
- Это устраняет проблемы с округлением и clamp-ом
- Для значений >= 1.0 использовать обычную формулу GF4

### 2. Экспериментальное исследование (Experimental Research)

**Рекомендуемые форматы для тестирования:**
1. **GF6A** — Тестировать φ-оптимизацию (PHI_BIAS=1) против стандартного GF6A
2. **GF4A-Adapt** — Сравнить адаптивный подход с обычным GF4
3. **GF5** — Для исследования нестандартных разбиений экспонента

**Ожидаемые результаты (Expected Outcomes):**
- GF6A-Φ может показать лучшую точность для значений в [1.0, 3.5] благодаря PHI_BIAS
- GF4A-Adapt может показать эффективность адаптивного подхода

### 3. Модульная интеграция (Module Integration)

**Требуемые изменения (Required Changes):**
1. Добавить `pub use experimental::*` в `mod.rs`
2. Обновить тесты для включения новых форматов
